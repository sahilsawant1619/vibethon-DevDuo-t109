Understanding the Perceptron

The perceptron is the fundamental building block of neural networks. It's a linear classifier that learns weights to make predictions.

Perceptron Implementation

import numpy as np

class Perceptron:
    def __init__(self, learning_rate=0.01, epochs=1000):
        self.learning_rate = learning_rate
        self.epochs = epochs
        self.weights = None
        self.bias = None
    
    def fit(self, X, y):
        # Initialize weights and bias
        n_samples, n_features = X.shape
        self.weights = np.zeros(n_features)
        self.bias = 0
        
        # Training loop
        for epoch in range(self.epochs):
            for i in range(n_samples):
                # Forward pass
                linear_output = np.dot(X[i], self.weights) + self.bias
                prediction = 1 if linear_output > 0 else 0
                
                # Update weights
                error = y[i] - prediction
                self.weights += self.learning_rate * error * X[i]
                self.bias += self.learning_rate * error
    
    def predict(self, X):
        linear_output = np.dot(X, self.weights) + self.bias
        return np.where(linear_output > 0, 1, 0)

# Example usage
X = np.array([[1, 1], [1, 0], [0, 1], [0, 0]])
y = np.array([1, 1, 1, 0])  # AND gate

perceptron = Perceptron()
perceptron.fit(X, y)
predictions = perceptron.predict(X)
print(f"Predictions: {predictions}")

Multi-Layer Perceptron (MLP)

Stack multiple perceptrons to create powerful non-linear models. The magic happens with activation functions and hidden layers.

Input Layer

Raw features fed into the network

Hidden Layers

Non-linear transformations using activation functions

Output Layer

Final predictions or classifications

Simple MLP Implementation

class MLP:
    def __init__(self, input_size, hidden_size, output_size):
        # Initialize weights with random values
        self.W1 = np.random.randn(input_size, hidden_size)
        self.b1 = np.zeros(hidden_size)
        self.W2 = np.random.randn(hidden_size, output_size)
        self.b2 = np.zeros(output_size)
    
    def sigmoid(self, x):
        return 1 / (1 + np.exp(-x))
    
    def sigmoid_derivative(self, x):
        return x * (1 - x)
    
    def forward(self, X):
        # Forward propagation
        self.z1 = np.dot(X, self.W1) + self.b1
        self.a1 = self.sigmoid(self.z1)
        self.z2 = np.dot(self.a1, self.W2) + self.b2
        self.a2 = self.sigmoid(self.z2)
        return self.a2
    
    def predict(self, X):
        predictions = self.forward(X)
        return np.where(predictions > 0.5, 1, 0)

Activation Functions

Activation functions introduce non-linearity, allowing neural networks to learn complex patterns.

Sigmoid

sigmoid(x) = 1 / (1 + e^(-x))

Range: (0, 1). Good for binary classification.

ReLU

ReLU(x) = max(0, x)

Range: [0, +inf). Most popular for hidden layers.

Tanh

tanh(x) = (e^x - e^(-x)) / (e^x + e^(-x))

Range: (-1, 1). Zero-centered, good for hidden layers.

A single perceptron can only learn linearly separable patterns. XOR is not linearly separable. You need at least one hidden layer (2-layer network) to solve XOR. This is why multi-layer networks are so powerful!

Backpropagation Algorithm

The engine that trains neural networks by efficiently computing gradients and updating weights.

The Chain Rule in Action

Backpropagation uses the chain rule to compute how each weight affects the final error.

Step 1: Forward Pass

Compute outputs layer by layer

a^(l) = f(z^(l)) where z^(l) = W^(l) * a^(l-1) + b^(l)

Step 2: Compute Error

Calculate loss at the output

L = 1/2 * sum((y_true - y_pred)²)

Step 3: Backward Pass

Propagate error gradients backward

delta^(l) = (W^(l+1))^T * delta^(l+1) * f'(z^(l))

Step 4: Update Weights

Adjust weights using gradients

W^(l) = W^(l) - learning_rate * delta^(l) * (a^(l-1))^T

Gradient Descent Variants

Different ways to update weights based on computed gradients.

Batch Gradient Descent

Uses entire dataset for each update. Stable but slow for large datasets.

W = W - learning_rate * grad(L, W)

Stochastic Gradient Descent

Updates after each sample. Fast but noisy convergence.

W = W - learning_rate * grad(L_i, W)

Mini-Batch GD

Best of both worlds. Uses small batches (32-256 samples).

W = W - learning_rate * grad(L_batch, W)

Complete Backpropagation Implementation

class NeuralNetwork:
    def __init__(self, input_size, hidden_size, output_size, learning_rate=0.01):
        self.W1 = np.random.randn(input_size, hidden_size)
        self.b1 = np.zeros(hidden_size)
        self.W2 = np.random.randn(hidden_size, output_size)
        self.b2 = np.zeros(output_size)
        self.learning_rate = learning_rate
    
    def sigmoid(self, x):
        return 1 / (1 + np.exp(-x))
    
    def sigmoid_derivative(self, x):
        return x * (1 - x)
    
    def forward(self, X):
        # Forward pass through network
        self.z1 = np.dot(X, self.W1) + self.b1
        self.a1 = self.sigmoid(self.z1)
        self.z2 = np.dot(self.a1, self.W2) + self.b2
        self.a2 = self.sigmoid(self.z2)
        return self.a2
    
    def backward(self, X, y, output):
        m = X.shape[0]
        
        # Calculate gradients
        d_output = (output - y) * self.sigmoid_derivative(output)
        d_hidden = d_output.dot(self.W2.T) * self.sigmoid_derivative(self.a1)
        
        # Update weights
        self.W2 -= self.learning_rate * self.a1.T.dot(d_output) / m
        self.b2 -= self.learning_rate * np.sum(d_output, axis=0) / m
        self.W1 -= self.learning_rate * X.T.dot(d_hidden) / m
        self.b1 -= self.learning_rate * np.sum(d_hidden, axis=0) / m
    
    def train(self, X, y, epochs):
        for epoch in range(epochs):
            output = self.forward(X)
            self.backward(X, y, output)
            
            if epoch % 1000 == 0:
                loss = np.mean((y - output) ** 2)
                print(f"Epoch {epoch}, Loss: {loss:.4f}")

Convolutional Neural Networks

Specialized neural networks for image processing and computer vision tasks.

Convolution Operation

The core operation that makes CNNs special. Apply filters to detect patterns like edges, textures, and shapes.

Convolution Layer

Applies learnable filters to input. Each filter detects specific patterns.

def convolution(image, kernel):
    # Simple 2D convolution
    height, width = image.shape
    k_height, k_width = kernel.shape
    
    # Calculate output size
    out_height = height - k_height + 1
    out_width = width - k_width + 1
    output = np.zeros((out_height, out_width))
    
    # Perform convolution
    for i in range(out_height):
        for j in range(out_width):
            region = image[i:i+k_height, j:j+k_width]
            output[i, j] = np.sum(region * kernel)
    
    return output

Pooling Layer

Reduces spatial dimensions while preserving important features.

def max_pooling(image, size=2, stride=2):
    height, width = image.shape
    out_height = (height - size) // stride + 1
    out_width = (width - size) // stride + 1
    output = np.zeros((out_height, out_width))
    
    for i in range(out_height):
        for j in range(out_width):
            region = image[i*stride:i*stride+size, 
                         j*stride:j*stride+size]
            output[i, j] = np.max(region)
    
    return output

Key CNN Concepts

Local Receptive Fields

Each neuron only sees a small region of the input, preserving spatial information.

Parameter Sharing

Same filter is used across entire image, reducing parameters dramatically.

Translation Invariance

Pooling makes the network robust to small translations of features.

Feature Hierarchy

Early layers detect edges, middle layers detect shapes, deep layers detect objects.

Simple CNN Implementation

class SimpleCNN:
    def __init__(self):
        # Initialize random filters (in practice, these are learned)
        self.conv1_filters = np.random.randn(8, 3, 3, 3) * 0.1
        self.conv2_filters = np.random.randn(16, 8, 3, 3) * 0.1
        self.fc_weights = np.random.randn(16 * 5 * 5, 10) * 0.1
    
    def relu(self, x):
        return np.maximum(0, x)
    
    def conv_forward(self, x, filters):
        # Simplified convolution forward pass
        batch_size, height, width, channels = x.shape
        n_filters, f_channels, f_height, f_width = filters.shape
        
        out_height = height - f_height + 1
        out_width = width - f_width + 1
        output = np.zeros((batch_size, out_height, out_width, n_filters))
        
        for i in range(batch_size):
            for f in range(n_filters):
                for h in range(out_height):
                    for w in range(out_width):
                        region = x[i, h:h+f_height, w:w+f_width, :]
                        output[i, h, w, f] = np.sum(region * filters[f])
        
        return self.relu(output)
    
    def max_pool_forward(self, x, size=2):
        batch_size, height, width, channels = x.shape
        out_height = height // size
        out_width = width // size
        output = np.zeros((batch_size, out_height, out_width, channels))
        
        for i in range(batch_size):
            for c in range(channels):
                for h in range(out_height):
                    for w in range(out_width):
                        region = x[i, h*2:h*2+2, w*2:w*2+2, c]
                        output[i, h, w, c] = np.max(region)
        
        return output
    
    def forward(self, x):
        # Forward pass through CNN
        conv1 = self.conv_forward(x, self.conv1_filters)
        pool1 = self.max_pool_forward(conv1)
        conv2 = self.conv_forward(pool1, self.conv2_filters)
        pool2 = self.max_pool_forward(conv2)
        
        # Flatten and classify
        flattened = pool2.reshape(pool2.shape[0], -1)
        logits = np.dot(flattened, self.fc_weights)
        
        return logits

Exercise: Why do CNNs work so well for images but not as well for text data? What's the key difference?

CNNs exploit spatial locality and translation invariance in images. Text has sequential structure and word order matters more than spatial position. For text, RNNs/Transformers are better suited as they can capture sequential dependencies and long-range relationships.
